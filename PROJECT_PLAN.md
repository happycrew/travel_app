# Aifory Stay — план продукта

## Этап 1. Лендинг и проверка спроса

- Уточнить позиционирование, географию запуска, поставщика отелей и сценарий оплаты.
- Собрать дизайн-систему: типографика, цвета, сетка, компоненты, motion-принципы.
- Реализовать mobile-first лендинг, поиск, подборки, FAQ, состояния загрузки и ошибок.
- Подключить SEO: метаданные, Open Graph, schema.org, sitemap, robots.txt.
- Подключить GA4/Яндекс Метрику через GTM, CMP/согласия, UTM и сквозные события.
- Целевые события: поиск, выбор направления, просмотр отеля, начало брони, способ оплаты, успешная оплата.
- Проверить Core Web Vitals, доступность WCAG 2.2 AA, адаптивность и браузеры.

## Этап 2. MVP бронирования

- Next.js App Router + TypeScript; API/BFF отдельно от интеграции поставщика.
- Поиск по городу и датам, выдача, фильтры, карточка отеля, выбор номера.
- Авторизация по email/телефону, профиль, список бронирований и ваучер.
- Интеграция hotel inventory API, нормализация цен, валют и условий отмены.
- Создание заказа с идемпотентностью, резервирование, подтверждение и отмена.
- Оплата криптовалютой и резервный фиатный сценарий, webhook-подтверждения.
- Email/мессенджер-уведомления и поддержка.

## Этап 3. Надёжность и рост

- PostgreSQL, Redis-кеш, очередь задач, audit log, Sentry/OpenTelemetry.
- Админка: заказы, возвраты, маржа, промокоды, контент и обращения.
- Антифрод, rate limiting, secrets management, бэкапы, роли и 2FA.
- A/B-тесты оффера и поиска, CRM-сегменты, abandoned booking и ретаргетинг.
- SSR/ISR для SEO-страниц направлений, CDN изображений, performance budget.
- Юридические документы, оферта, политика данных, PCI/KYC/AML по платёжной модели.

## Рекомендуемые KPI

- Search start rate, search → hotel view, hotel view → checkout.
- Checkout → paid booking, ошибки поставщика/оплаты, время подтверждения.
- CAC, ROAS, валовая маржа, повторные брони, отмены и обращения в поддержку.
- LCP ≤ 2,5 c, INP ≤ 200 мс, CLS ≤ 0,1 на p75 мобильного трафика.
